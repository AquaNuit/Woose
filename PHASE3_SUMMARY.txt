🔐 GITHUB SECRETS CONFIGURATION
═══════════════════════════════════════════════════════════════════════════════

Required Secrets (7 total):

1. BUILD_CERTIFICATE_BASE64
   • Source: Certificates.p12 exported from Keychain or OpenSSL
   • Format: Base64-encoded PKCS#12 file (~2800 characters)
   • Generated: base64 Certificates.p12 > cert_base64.txt
   • Used by: Certificate installation step in CI/CD

2. P12_PASSWORD
   • Source: Password you set when exporting certificate
   • Format: Plain text string
   • Security: Store in password manager
   • Used by: security import command in CI/CD

3. BUILD_PROVISION_PROFILE_BASE64
   • Source: BevelClone_Development.mobileprovision from Apple
   • Format: Base64-encoded XML plist (~8000 characters)
   • Generated: base64 BevelClone_Development.mobileprovision > pp_base64.txt
   • Used by: Provisioning profile installation step

4. WATCH_PROVISION_PROFILE_BASE64 (Optional for Phase 3)
   • Source: Watch app provisioning profile
   • Format: Base64-encoded XML plist
   • Note: Can be added later when Watch target is fully configured

5. KEYCHAIN_PASSWORD
   • Source: Generate random strong password
   • Format: Plain text (e.g., "Temp$Keychain2024!")
   • Purpose: Temporary keychain created during CI build
   • Security: Only exists for duration of workflow

6. DEVELOPMENT_TEAM
   • Source: developer.apple.com → Account → Membership
   • Format: 10-character alphanumeric (e.g., "A1B2C3D4E5")
   • Used by: CODE_SIGN_IDENTITY and DEVELOPMENT_TEAM in xcodebuild

7. PROVISIONING_PROFILE_NAME
   • Source: Exact name from developer.apple.com → Profiles
   • Format: String matching profile (e.g., "Bevel Clone Development")
   • Used by: PROVISIONING_PROFILE_SPECIFIER in xcodebuild
